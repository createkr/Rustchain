Vulnerability #3: Clickjacking / UI Overlay

The widget modal has z-index: 999999 with no frame-busting protection. A malicious site can embed the widget in an iframe and overlay a transparent button to trick users into signing payments.

⚠️ This PoC shows the vulnerability pattern. Full attack requires the widget to be embedded in an iframe on a malicious page.

The attack flow:

  1. Attacker embeds widget in iframe on evil.com
  2. Attacker overlays a transparent "Confirm Payment" button at z-index > 999999
  3. Victim thinks they're clicking "Cancel" but actually triggers payment
iframe {
  opacity: 0.01;        /* almost invisible */
  position: absolute;
  top: 0; left: 0;
  z-index: 9999999;     /* above widget */
}
button {
  position: absolute;
  top: 620px; left: 80px; /* on top of "Sign & Send" button */
  width: 200px; height: 48px;
  opacity: 0.01;
}

Fix: Add blockFraming() at the top of openPaymentModal() to prevent iframe embedding entirely.
