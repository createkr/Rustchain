Vulnerability #1: Stored XSS via data-memo / data-to

The openPaymentModal() function in rustchain-pay.js injects config.memo and config.to directly into innerHTML without sanitization: 

overlay.innerHTML = `
  <p>Memo: ${config.memo}</p>   ← UNSAFE
  <p>To: ${config.to}</p>       ← UNSAFE
`;

To exploit, host this HTML file and include the widget. When the victim opens the payment modal, the onerror fires:

Steps to reproduce:

  1. Open this HTML file in a browser
  2. Click the orange "Pay 1 RTC" button
  3. Observe: alert('XSS via memo!') fires immediately (Vuln #1)
  4. In the textarea, type any seed phrase
  5. Observe: The seed phrase appears in the "SEED STOLEN" div (simulation of theft)

Real attack: attacker POSTs stolen seed to https://attacker.com/steal
